## 한 줄 서사
딥다이브하는 5년차 시스템 엔지니어가 블록체인으로 확장한다.
Android 오디오 커널~프레임워크에서 판 저수준·동시성·성능·설계판단 감각을,
reth 코어 분석과 프로토콜 검증으로 증명한다. 동시에, 블록체인 인프라를
사용자 관점에서 다시 보는 UX 감각으로 이 판의 진짜 병목(복잡성 은닉)을 짚는다.
 
## 나는 누구인가 (강점 = 근거)
- **저수준 시스템·동시성** — Samsung MX에서 Android Audio Framework(HAL~App) 설계·유지보수 4~5년.
  AudioFlinger BufferProvider 계층 설계, HIDL→AIDL 전환 중 audioserver 안정화·커널 패닉 근본 원인 분석.
  → 락/스레드/공유상태를 코드 레벨에서 다뤄온 사람.
- **성능 감각(수치 기반)** — VoIP Live Translation underrun 노이즈를 4개 레이어 동시 로그 분석으로 해결,
  USB 탈부착 routing latency를 수치 기반 보정. → "규모 커지면?" "최적화는?"에 데이터로 답하는 사람.
- **설계 판단·트레이드오프** — File Capsule(Tauri+Rust, 3-OS 동시 출시)에서 Arc<Mutex> 공유상태 →
  메시지 패싱 액터 모델로 전환, 설계 오류로 4,000줄 두 번 폐기, Severity(Level 1~4) 기반 기술 토론 체계 수립.
  → "왜 이렇게 했어요?"에 대안·트레이드오프로 답하는 사람.
- **Rust/tokio 실전** — File Capsule 액터 아키텍처(SearchIndexerActor, BoardWatcherActor),
  크로스플랫폼 FS 이벤트 순서 차이 흡수, 성능 목표("한 글자 검색 20,000건 3초 내")에서 역산한 아키텍처 설계.
  → reth(Rust)로 자연스럽게 넘어가는 다리.
- **UX·시스템 사고의 교차점** — 크립토 인프라(멀티체인 지갑의 "네트워크 골라" 화면)를
  leaky abstraction으로 읽어내는 감각. 기술을 아는 사람 중에 드물게, "이 복잡성이 왜 유저에게
  새어나오는가"를 인터페이스 설계 언어로 짚을 수 있음.
## 블록체인으로 뭘 가져가나 (매핑)
- AudioFlinger 락/동시성 사고 → reth txpool/state 동시성 분석
- 오디오 성능·엣지케이스 감각 → 가스 최적화·공격 시나리오·엣지 테스트
- File Capsule 설계 판단(대안 비교, 폐기 결정) → 프로토콜 설계 트레이드오프 읽기
- 다층 로그로 근본 원인 추적하는 습관 → 백서 주장 vs 소스 대조 검증
- UX 감각(leaky abstraction 식별) → 프로토콜/지갑 UX 병목 진단, 온보딩 설계 논의에 기여
## 이미 있는 블록체인 근거
- **zkbench** — Noir 회로 증명 벤치마커, crates.io 공개
- **ZK Voting** — 머클/nullifier 회로 (Phase 3에서 믹서로 확장)
- **velog 0편** — "탈중앙을 외치는데, 왜 자꾸 중앙으로 모일까": GENIUS Act 이자 금지의 우회 구조,
  DAI의 USDC 담보 재중앙화, 한국 51%룰의 은행법 충돌까지 다룬 도메인 분석 글.
  기술 구현 이전에 이미 판 전체 구조를 자기 언어로 정리한 상태.
- 블록체인 개념을 C++/Rust 멘탈모델로 매핑하는 습관 (예: DAI 청산=오디오 언더런 원인 분석과
  같은 "왜 실패하는가"를 근본까지 추적하는 방식)
## 목표 포지션 & 타깃
- 포지션: Rust·프로토콜 심층 리서치/코어 엔지니어
- 타깃: 국내 블록체인/스테이블코인 기술기업(빗썸·두나무 등) + 크립토 전반
  (원화 스테이블코인 법제화 타이밍과 무관하게, durable 스킬로 지원 가능하게 준비)
- 시점: 6개월 후부터 지원 시도 가능
- 이 전환은 "재밌어서 깊게 판다"가 1차 동기 — 토스나 특정 회사를 향한 즉시 피벗이 아니라
  병행 트랙. 그래서 산출물의 기준은 "이직 성공"이 아니라 "내가 이 판을 얼마나 깊이 이해했나"에 둔다.
  단, 실무면접 경쟁력(설계문서·트러블슈팅·꼬리질문 대응)은 의식적으로 같이 챙긴다.
## 정직한 갭 (숨기지 말고 프레이밍)
- 블록체인 도메인 지식은 배우는 중(신규). 단, **엔지니어링 성숙도는 5년차** —
  신입이 못 만드는 깊이로 그 갭을 덮는다.
- 담보형(DAI류) 스테이블코인을 Phase 1에서 직접 만들지만, 시장은 예치금형(USDC류)이 이긴다는 걸
  이미 알고 시작한다. 모순이 아니라 의도: "만들 줄도 알고, 왜 시장이 다른 걸 골랐는지도 안다"는
  포지션을 위한 선택.
